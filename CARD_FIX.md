# 数据卡片交互优化

## 问题
之前的数据卡片在鼠标移出地图区域后会立即消失，导致用户无法点击卡片中的"进入城市孪生"按钮。

## 解决方案

### 1. 添加卡片悬停状态
- 新增 `isCardHovered` 状态来跟踪鼠标是否在卡片上
- 新增 `cardHideTimer` 定时器来延迟隐藏卡片

### 2. 卡片鼠标事件
给数据卡片添加了两个事件监听：
- `@mouseenter="onCardMouseEnter"` - 鼠标进入卡片时
- `@mouseleave="onCardMouseLeave"` - 鼠标离开卡片时

### 3. 延迟隐藏机制
- 当鼠标离开地图对象时，不立即隐藏卡片，而是设置200ms延迟
- 如果用户在延迟时间内将鼠标移到卡片上，取消隐藏
- 当鼠标离开卡片时，再设置300ms延迟隐藏

### 4. 样式优化
- 确保卡片的 `pointer-events: auto` 可以接收鼠标事件
- 添加 `cursor: default` 让鼠标在卡片上显示正常光标

## 使用体验

现在用户可以：
1. 鼠标悬停在地图上的市/区域，显示数据卡片
2. 将鼠标从地图移动到数据卡片上（卡片不会消失）
3. 点击"进入城市孪生"按钮
4. 点击关闭按钮或将鼠标移开一段时间后，卡片自动隐藏

## 技术细节

**延迟时间设置：**
- 地图移出 → 卡片隐藏：200ms
- 卡片移出 → 卡片隐藏：300ms

这些延迟时间经过优化，既不会让卡片消失太快影响用户操作，也不会让卡片停留太久影响视觉体验。

**清理机制：**
- 在组件卸载时清理所有定时器，防止内存泄漏
- 手动关闭卡片时立即清理定时器
